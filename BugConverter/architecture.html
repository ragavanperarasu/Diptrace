③ Intelligence Layer — Thinking Model + System-Controlled RAG
Primary Pipeline — LLM Reasoning
Session Init — Dynamic Protocol Generation
LLM receives patient history + visit reason → generates personalized
prescreening protocol. No static checklists — clinically reasoned per patient.
↓
Pass 1 — Reasoning + Retrieval Decision
LLM processes utterance chunk against: running context (sliding window
+ extracted facts), patient history, dynamic protocol.
Outputs: clinical entities, protocol update, retrieval queries (if needed),
preliminary suggestions, flags.
↓ retrieval queries (if triggered)
System-Controlled RAG Retrieval
Orchestrator executes retrieval in parallel (not agentic).
Drug KB: metadata filter → vector search
Clinical Guidelines KB: semantic search with body system filter
Learning KB: matched rules by conditions/medications
↓ enriched context
Pass 2 — Enriched Reasoning (only when retrieval triggered)
LLM reasons with retrieved knowledge + all prior context.
Outputs: refined suggestions, clinical flags, contradiction alerts,
updated dynamic protocol, new protocol items if emerging concerns.

⚡ Latency Budget
  • Single pass (no retrieval): ~1-2 seconds
  • Two-pass (with retrieval): ~3-5 seconds
  • Retrieval runs in parallel while LLM completes Pass 1
  • ~70-80% of chunks single-pass only
  • Displayed with 15-20s delay for completeness monitoring
✓ V1 Design Decisions
  • Completeness monitoring over real-time guidance
  • High-value interventions only (flags, critical gaps)
  • Doctor feedback via UI, not speech capture
  • Utterance-based processing, not time-based
  • Nurse correction feedback on suggestions
  • No multimodal audio analysis yet
⚙ RAG Strategy
  • Drug KB: exact name filter → vector search (hybrid)
  • Guidelines KB: semantic search + metadata filter
  • Learning KB: match by conditions, medications, demographics
  • Retrieval triggered by LLM, executed by system orchestrator
  • Max 2-3 lookups per chunk, bounded by orchestrator
↻ Learning Safeguards
  • Confidence tiers: new → emerging (2x) → established (3x+)
  • Clinical admin approval required for established rules
  • No negative learning (never learns to skip screening)
  • Doctor attribution on feedback
  • Quarterly review cycle with decay

△ V1 → V2 Changes
  • Completeness monitoring → proactive real-time guidance
  • Added multimodal audio analysis (supplementary layer)
  • Added doctor speech capture for automated feedback
  • Behavioral observations in summary for physician
  • Enhanced consent for voice behavioral analysis
  • Behavioral data store with separate retention
  • Multi-clinic knowledge sharing capability
🎙 Multimodal Layer Design
  • 60-90 second batched segments (not real-time)
  • Runs parallel — never blocks primary pipeline
  • Running emotional state carries across segments
  • Observations only, never diagnostic claims
  • Nurse can override any behavioral signal
  • ~13-20 API calls per 20-min consultation
↻ Dual Feedback Channels
  • UI feedback: explicit, structured, immediate
  • Speech capture: implicit, richer, requires extraction
  • 3-speaker diarization for doctor consultations
  • Gap detection by comparing doctor Qs vs summary
  • Both channels merge into unified learning pipeline
  • Behavioral signal validation from doctor follow-through
⚠ Privacy & Ethics (V2 Enhanced)
  • Enhanced consent: disclose behavioral voice analysis
  • Behavioral data has separate, shorter retention
  • System observes, never diagnoses emotional states
  • Nurse override always authoritative over AI analysis
  • No cross-patient behavioral profiling
  • Doctor speech: consent from all parties required
